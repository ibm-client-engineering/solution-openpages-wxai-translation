## Method 2: Custom Translation Service
-----
### Where it's used

1. Navigate to any page within OpenPages where the translation button appears.

    ![translation button](https://media.github.ibm.com/user/386696/files/9c44f1f6-3498-4c83-9c41-58ec3995f481)

2. Depening on what locale the user is in, the button will translate the fields to that target locale language.
    
    *Example below w/ japanese locale*
    ![translation button 2](https://media.github.ibm.com/user/386696/files/8a278b7f-f1b7-4457-b379-8409046e128a)

### How to Implement

To translate the necessary headers/labels with wx.ai an API wrapper is required to simulate the API response that is expected from the NeuralSeek configuration within OpenPages.

Navigate [here](https://cloud.ibm.com/resources) for a walkthrough on how to implement the API wrapper.
